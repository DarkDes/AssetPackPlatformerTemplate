--------------------------
ОБЩЕЕ
--------------------------
PNG с прозрачностью. ВСЕ спрайты могут иметь анимацию, кроме тайловой карты. Размер указан с множителем 1 (для «пиксель арта»), множитель может быть до 3. Имеется таблица переходов на картинке выше. Например, спрайт монеты 16×16, если вы рисуете пиксель арт, а если рисуете HD, то возьмите размер 48×48. В первом случае в настройках ассет пака укажите размерность 3, а во втором — 1.


--------------------------
ИМЕНА ФАЙЛОВ И ДИРЕКТОРИЙ
--------------------------
С игрой идёт папка Assets, где имеются примеры от меня. Оформлять нужно так: /Assets/*ваш никнейм*/*название ассет пака*/*тут содержимое*.
Никнейм и название пака могут быть “техническими” т.е. без пробелов, без заглавных букв: в настройках можно будет указать красивое имя для отображения. Смотрите пример Assets/darkdes/antivirus.
Как указано на картинке выше имена файлов ассетов строятся следующим образом: 
*имя спрайта*_*анимация*_*кадр*.png.
Например: enemy_move_0.png
Имена спрайтов можно посмотреть в примерах или же ниже в описании оно будет в скобках + там же пример именования.


--------------------------
АССЕТ ПАК ДОЛЖЕН ВКЛЮЧАТЬ:
--------------------------
1) Главный персонаж игрока (player)
Пример: Марио, Соник, Самус, тысячи их. Это может быть и простой шар.
Размер 32х32
Анимации крайне желательно:
idle - ожидание, цикличная
move - движение, цикличная
jump - небольшая анимация когда персонаж в прыжке
fall - небольшая цикличная анимация когда персонаж падает
action - НЕ цикличная анимация действия, ОПЦИОНАЛЬНО
Пример имени файла: player_idle_0.png

2) Основной подбираемый предмета (coin)
Пример: монетки из Марио, кольца из Соника.
Размер 16х16
Пример имени файла: coin_1.png

3) Враг (enemy)
Например: Гумба из Марио. Наносит урон прикосновением. Убивается прыжком игрока сверху на голову врага.
Размер 32x32
Анимации:
move - передвижение, цикличная
dying - одноразовая, остановка на последнем кадре
Пример имени файла: enemy_dying_2.png

4) Стационарная опасность (danger)
Например: шипы, кусок лавы, циркулярная пила.
Размер 32х16
Может иметь анимацию.
Пример имени файла: danger_3.png

5) Тайлсет из 15 тайлов (tileset_0)
Тайлы “земли” по которой будет ходить игрок и враги. Самый минимальный набор.
Размер изображения: 80x80 для pixel art или 240x240 для HD. Размер одного тайла 16х16 или 48х48 соответственно.
Имя файла: tileset_0.png. Не ошибка, должен иметь именно такое имя с цифрой 0, не может иметь анимации.

6) Платформа стандартная (platform_static) [Можно без анимации]
Игрок может наступать сверху, снизу она пропускает игрока в прыжке. Можно считать это дополнительными тайлами/ Лучше использовать отдельный дизайн, чтобы отличалось от “земли”.
Размер 48x16
Пример имени файла: platform_static_5.png

7) Платформа подвижная (platform_move) [Можно без анимации]
В отличии от стандартной платформы может двигаться между двумя точками. Дизайн должен отличаться от стандартной и “земли”.
Размер 48x16
Пример имени файла: platform_move_6.png

8) Финиш уровня (flag) [Можно без анимации]
Например: флагшток в Марио. Объект, при котором завершается уровень.
Размер 32x32
Может иметь анимацию.
Пример имени файла: flag_7.png

9) Задний фон уровня (background) [Можно без анимации]
Фон всегда рисуется и неподвижен.
Размер 320x180
Пример имени файла:background_8.png

10) Фон параллаксный (background_parallax) [Опционально; Можно без анимации]
Фон движется с определённым коэффициентом параллаксом. Должен иметь прозрачные элементы, чтобы был виден основной фон.
Размер 320x180.
Пример имени файла:background_parallax_9.png

11) Декорация 1. Малая (decoration_small) [Опционально; Можно без анимации]
Например: Травинки.
Размер 16x16
Пример имени файла: decoration_small_10.png

12) Декорация 2. Средняя (decortaion_mid) [Опционально; Можно без анимации]
Например: Кусты.
Размер 32x32
Пример имени файла: decortaion_mid_11.png

13) Декорация 3. Большая (decoration_big) [Опционально; Можно без анимации]
Например: Дерево.
Размер 48x48
Пример имени файла: decoration_big_12.png

14) UI: Иконка индикатора жизни (ui_live) [Можно без анимации]
Например: сердца в Legend of Zelda, лицо Соника. Элемент интерфейса, показатель жизни. Может быть настроен на показ количеством (напр, три сердца) или цифровой показатель (напр, сердце и рядом цифра 3).
Размер 16x16 
Пример имени файла: ui_live_13.png

15) UI: Иконка индикатора подбираемого предмета (ui_coin) [Можно без анимации]
Например: кольцо на интерфейсе в играх про Соника.
Элемент интерфейса, показывает количество подобранных предметов.
Размер 16x16
Пример имени файла: ui_coin_14.png

16) Мокап т.е. скриншот собранный из нарисованной графики.
Размер 320x180. Или 960x540.
Содержит фейковый скриншот из подготовленной графики. Или достаточно сделать скриншот из игры. Будет "обложкой" вашей заявки.


--------------------------
ASSET PACK PLATFORMER TEMPLATE (APPT) 
--------------------------
это игра и одновременно инструмент.
Управление в игре:
AD, стрелка влево-вправо, D-Pad влево-вправо, Левый стик влево-вправо – движение влево и вправо соответственно.
Space, W, стрелка вверх, Z, Gamepad A - прыжок
Ctrl, X, Gamepad X - действие (только анимация)
R - Рестарт уровня
F1 - Включить DevTool

--------------------------
DEVTOOL
--------------------------
Пояснения интерфейса APPT. При включении слева сверху появляется интерфейс с элементами: название текущего ассет пака, кнопка редактирования (EDIT), чекбокс пропуска заставки-представления ассет пака (SKIP INTRO) и кнопка добавления нового ассет пака (ADD NEW).

При нажатии ADD NEW появляется диалог создания нового ассет пака, куда вводится ИМЯ ПАПОК.

При нажатии на EDIT открываются настройки текущего выбранного ассет пака.

1 ) ASSET PACK NAME это собственно визуальное наименование ассет пака, которое показывается.
2 ) AUTHOR - визуальное отображение имени автора.
3 ) PIXEL ART - если включен, то спрайты пиксельные, иначе всё размыто. Также известен как линейная фильтрация.
4 ) DEFAULT FPS - базовое значение кадров в секунду для всех спрайтов.
5 ) SPRITE SCALE - множитель умножения размера спрайтов.
6 ) UI LIVES DISPLAY - отображение количества жизней. Если указано COUNT, то рисуется несколько раз спрайт жизней, а если NUMBERS, то показывается текстом количество.
7 ) UI COINS DISPLAY - отображение количества “монет”. Значение и логика такая же, как и у UI LIVES DISPLAY.
8 ) Далее Список спрайтов (SPRITES:) кликая на каждый меняется список анимаций (ANIMATIONS), где элементы тоже кликабельны. И кликнув на анимацию доступна одна настройка  FPS т.е. количество кадров в секунду. Можно удалить настройку у анимации через кнопку DELETE DATA, чтобы использовалась DEFAULT FPS. 
9 ) Кнопка SAVE сохраняет изменения настроек в файл SETTINGS.JSON.
10) Кнопка RELOAD подгружает файл SETTINGS.JSON и применяет настройки из него. Это похоже на сброс настроек.

Файл SETTINGS.JSON имеет простой вид ключей и значений. Вы можете редактировать этот файл, например, в блокноте, если нужно, но имейте ввиду, что APPT не подхватит изменения автоматически – нужно будет нажать кнопку RELOAD. Но я не рекомендую этого делать – лучше всё делать через интерфейс.
Ключ кадров в секунду для спрайтов такой: sprites_*имя спрайта*_*имя анимации или idle*_fps.
Пример файла SETTING.JSON:
{
	"assetpack_name": "Anti-Virus Platformer",
	"default_fps": 15,
	"pixel_art": true,
	"sprite_scale": 3,
	"sprites_background_idle_fps": 19,
	"sprites_background_parallax_idle_fps": 15,
	"sprites_coin_idle_fps": 15,
	"sprites_danger_idle_fps": 15,
	"sprites_decoration_big_idle_fps": 15,
	"sprites_decoration_mid_idle_fps": 15,
	"sprites_decoration_small_idle_fps": 15,
	"sprites_enemy_dying_fps": 15,
	"sprites_enemy_move_fps": 15,
	"sprites_flag_idle_fps": 15,
	"sprites_platform_move_idle_fps": 15,
	"sprites_platform_static_idle_fps": 15,
	"sprites_player_fall_fps": 15,
	"sprites_player_idle_fps": 15,
	"sprites_player_move_fps": 15,
	"sprites_ui_coin_idle_fps": 15,
	"sprites_ui_live_idle_fps": 15,
	"ui_coins_method": "NUMBERS",
	"ui_lives_method": "COUNT"
}


--------------------------
Инструкция
--------------------------
И так, чтобы создать ассет пак вы можете скопировать один из шаблонов (template_pixel или template_hd) и отредактировать под свои нужды. Или же через игру нажать ADD NEW и ввести ИМЯ ПАПОК с вашим ником и названием ассет пака, НО тогда вам придётся сохранять и называть файлы спрайтов вручную по инструкции.

После того, как ваш ассет пак будет готов, то вы можете сделать скриншот специальной кнопкой и использовать его как обложку для вашей заявки.
